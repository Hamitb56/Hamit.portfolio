ReDaP: Reliable Datagram Protocol Design & Implementation

ReDaP is a lightweight, reliable transport layer protocol built on top of UDP, designed to bridge the gap between the speed of connectionless protocols and the reliability of TCP. This project involved defining a custom RFC specification and implementing a complete Sender/Receiver architecture in Python.

Technical Architecture & Protocol Specifications

  • Reliability Mechanism: Implemented a Stop-and-Wait ARQ mechanism, ensuring data integrity without overhead.
  • Error Detection: Developed a 16-bit one’s complement checksum algorithm to validate packets at the word level.
  • Packet Structure: Designed a custom 10-byte fixed header comprising Message Type, Flags, Sequence Numbers, and Checksum.
  • Sequencing & Flow: Utilized 16-bit sequence numbers to handle packet ordering and large file transmissions in 512-byte chunks.
  • Dynamic Retransmission: Engineered a timeout-based system to handle network congestion or packet drops effectively.

Software Implementation (Python)

  • Socket Programming: Developed asynchronous communication scripts using Python’s socket and struct libraries.
  • Performance Metrics: Tracks real-time statistics including throughput (KB/s) and retransmission counts.
  • User Interface: Integrated a tkinter file-dialog system for seamless file selection and transmission.
